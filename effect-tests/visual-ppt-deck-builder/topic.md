# 主题确认

## 测试主题

普通人如何用 Codex 把一个想法变成可交付的视觉方案

## 模拟确认结果

- 受众：不是纯技术背景的个人创作者、小团队负责人、课程/方案制作者。
- 使用场景：一场 8-10 分钟的内部分享或小型公开课，用来解释 Codex 不是“写代码工具”，而是可以把模糊想法推进成可交付视觉方案的协作工作台。
- 核心目标：让听众理解从一句想法到 PPT、图片素材、网页原型、交付包的完整路径。
- 语气：清醒、实用、有一点鼓舞感，不夸大 AI，不把流程说成玄学。
- 交付形态：可编辑 PPTX，正文和图表保持可编辑；允许使用少量视觉背景或装饰，但不把整页烤成截图。
- 风格倾向：选择“编辑杂志型 + 产品演示型”的混合方向，既适合讲故事，也能展示流程和成果。

## 约束

- 不使用真实客户数据。
- 图表数据只表达流程权重和示意，不伪装成事实统计。
- 只在本次效果测试目录下写入文件。
