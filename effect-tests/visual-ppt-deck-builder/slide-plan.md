# Slide Plan

总页数：8 页

## 1. 封面

- 标题：普通人如何用 Codex 把一个想法变成可交付的视觉方案
- 核心信息：把 Codex 定义成“从想法到交付包”的推进器。
- 证明对象：这不是聊完就结束，而是能产出文件、素材、说明和验收记录。
- 视觉：杂志式标题 + 右侧抽象工作台。

## 2. 痛点

- 标题：多数想法不是没价值，是没有被推进
- 核心信息：想法常卡在表达不清、风格不定、交付物缺失、验收标准模糊。
- 证明对象：普通人最缺的是连续推进机制。
- 视觉：左侧问题说明，右侧空白工作区。

## 3. 方法

- 标题：Codex 的作用：把模糊意图拆成可确认动作
- 核心信息：确认主题、大纲、风格、页面计划、素材、最终文件。
- 证明对象：流程本身会降低认知负担。
- 视觉：要点列表 + 右侧视觉框。

## 4. 六步流程

- 标题：从一句话到视觉方案的 6 个动作
- 核心信息：主题、故事、风格、页面、素材、交付。
- 证明对象：每一步都有产物，不靠临场发挥。
- 视觉：时间线。

## 5. 方式对比

- 标题：三种做法的差别
- 核心信息：传统做法成本高；纯聊天 AI 容易停在文字；Codex 工作流能落到文件。
- 证明对象：Codex 的差异在“执行闭环”。
- 视觉：三栏对比。

## 6. 人机分工

- 标题：普通人负责判断，Codex 负责推进
- 核心信息：人负责方向、取舍、审美判断；Codex 负责整理、生成、验证、沉淀。
- 证明对象：不是把创意外包，而是把摩擦交给工具。
- 视觉：内容页。

## 7. 时间投入示意

- 标题：一次小项目的时间应该花在哪里
- 核心信息：更多时间花在确认和验收，而不是重复搬运。
- 证明对象：流程化后，制作环节被压缩。
- 视觉：可编辑条形图，数据为示意。

## 8. 收束

- 标题：下一步：从一个小主题跑完整闭环
- 核心信息：不要从宏大项目开始，先用一个小主题跑通主题、风格、PPTX、素材、报告。
- 证明对象：完成一次，比收藏十个教程更有用。
- 视觉：结束页。
